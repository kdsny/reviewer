Core Programming Concepts & Definitions

Variable

Named memory location holding value of specific type.
  • Declaration: int age;
  • Initialization: age = 25;
  • Types: Primitive (int, double); Reference (String)
  • Scope: Local, instance, static

Data Types

Specifies value type a variable can hold.
  • Primitive: int, double, boolean, char, byte, short, long, float
  • Defaults: int=0, double=0.0, boolean=false
  • Sizes: int (4 bytes), double (8 bytes)

Array

Fixed-size collection of same-type elements.
  • Declaration: int[] numbers;
  • Creation: new int[5];
  • Index: Starts at 0
  • Length: numbers.length
  • 2D: int[][] matrix;

String

Immutable sequence of characters (class).
  • Creation: "John";
  • Methods: length(), charAt(), substring(), toUpperCase(), equals()
  • Pool: Literals shared in memory

Boolean

Primitive type: true or false.
  • Default: false
  • Operators: &&, ||, !
  • Usage: Conditions, logic

Loop

Repeats code block until condition met.
  • for: Fixed iterations
  • while: Pre-check condition
  • do-while: Post-check
  • Enhanced: For collections
  • Keywords: break, continue

Conditional Statement

Executes based on boolean condition.
  • if-else: Basic decisions
  • switch: Multi-branch
  • Ternary: ? :

Method (Function)

Reusable code block for task.
  • Syntax: returnType name(params)
  • void: No return
  • Static: Class-level
  • Overload: Same name, different params

Class & Object

Class: Blueprint; Object: Instance.
  • Class: class Person { }
  • Object: new Person();
  • Constructor: Initializes object
  • this: Current object

OOP Principles

Core OOP concepts.
  • Encapsulation: Data hiding
  • Inheritance: extends
  • Polymorphism: Multiple forms
  • Abstraction: Hide details

Exception

Event disrupting program flow.
  • try-catch: Handle
  • finally: Always run
  • throw/throws: Manual/declare
  • Types: Checked/unchecked

Database

Organized data collection.
  • DBMS: MySQL, Oracle
  • Table: Rows/columns
  • SQL: Query language
  • Keys: Primary, foreign

Algorithm

Step-by-step problem-solving procedure.
  • Structure: Input-process-output
  • Efficiency: Time/space complexity
  • Examples: Sort, search

IDE

Tool for coding/debugging.
  • Examples: IntelliJ, Eclipse
  • Features: Completion, debugger

Operator

Symbol for operations on values.
  • Arithmetic: +, -, *, /, %
  • Relational: >, <, ==, !=
  • Logical: &&, ||, !
  • Assignment: =, +=, -=

Inheritance

Mechanism for subclass to inherit from superclass.
  • Keyword: extends
  • Benefit: Code reuse
  • super: Call parent constructor/method

Polymorphism

Same interface for different underlying forms.
  • Overriding: Subclass redefines method
  • Overloading: Same name, different params
  • Runtime: Method binding at runtime

Encapsulation

Bundling data and methods; hiding internals.
  • Access modifiers: private, public
  • Getters/Setters: Control access

Abstraction

Hiding complexity, showing essentials.
  • Abstract class: Cannot instantiate
  • Interface: Pure abstraction

Interface

Contract for methods to implement.
  • Keyword: interface
  • Implements: Class keyword
  • Default methods: Since Java 8

Constructor

Special method to initialize object.
  • No return type
  • Overloading: Multiple constructors
  • Default: Provided if none defined

Recursion

Method calling itself.
  • Base case: Termination condition
  • Risk: Stack overflow
  • Example: Factorial

Big O Notation

Describes algorithm complexity.
  • O(1): Constant time
  • O(n): Linear
  • O(n^2): Quadratic

API

Interface for software interaction.
  • REST: Web API style
  • Endpoint: URL for access

Compiler

Translates code to machine language.
  • Java: javac to bytecode
  • Vs Interpreter: Ahead-of-time vs runtime

Interpreter

Executes code line by line.
  • Example: Python interpreter
  • Advantage: Faster development

Variable Scope

Region where variable is accessible.
  • Local: Method/block
  • Global: Class level

Type Casting

Converting one type to another.
  • Implicit: Widening (int to double)
  • Explicit: Narrowing (double to int)

Garbage Collection

Automatic memory management.
  • Java: JVM handles
  • Mark-Sweep: Common algorithm
